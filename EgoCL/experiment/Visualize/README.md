这个文件夹下是一个实验数据可视化模块，

包括三个模块
（1）数据加载模块
    （a）数据随机生成模块：生成随机实验数据用于测试和演示。实验数据形如：
        很多很多二维方形区域中的点，所有这些点都位于方形的右下侧的三角形区域中，
        每个点有一个值，取值范围在0到1之间。1被认为是好值，0被认为是坏值。
        方形的轴可以理解为时间，时间虽然基本单位是秒，但是表示的时候需要以小时：分钟：秒的格式显示。
        再生成随机数据的时候，需要服从，离对角线比较近的点值更大，离对角线比较远的点值更小的基本规律，在这样一个总体趋势下略有随机波动。
    
        此随机生成模块还需要支持将生成好的随机数据保存下来，还需要支持从保存下来的数据文件中读取数据而非重新随机生成数据。
    （b）数据加载模块：从实验数据文件中读取数据。
        不过实验还没有做，实验数据更是没有准备好，但其形态结构是如上面（a）中描述的那样了，
        你可以把读取程序写好，从而相当于把实验数据的存储格式也确定了，并写出一小段样例程序以表达存储代码的大概形态

（2）基于Pyplot的数据可视化模块
    （a）静态图片生成模块：使用matplotlib.pyplot等可视化库生成实验数据的静态图片。
    （b）在得到如上数据（本质是一个散点图）后，需要以热力图的形式可视化，热力图中的各个格子需要综合其周围散点的值进行平滑化后获得该格子上的颜色，
    （c）在这个三角形的左上方是一个空白区域，颜色条的图例就放在这个区域
    （d）这个三角形的横轴以及刻度在下方，纵轴以及刻度请设立在右方
    （e）在这个三角形的右方，还需要绘制一个横向的柱状图，表示每一段的“离对角线距离”中，所有点的平均值是多少；离对角线距离最大的一根柱子放在横向柱状图的最上方，最小的柱子放在最下方（结合上方给出的数据分布预期，这个柱状图应该是一个从上到下逐渐变短的趋势）
    （f）在柱状图的右方也请标注上对应的平均值，改用百分值（0.87的值写作87%），柱状图的横轴刻度请设立在下方

（3）基于echart的网页端数据可视化模块
    （a）本地服务器模块：使用fastapi等轻量级Web框架搭建本地服务器，提供数据可视化的网页端接口，并将特定的实验数据传递给网页进行展示。
    （b）前端展示模块：使用echart等前端可视化库，在网页端展示实验数据的可视化结果。
    （c）而展示的形式和（2）中描述的类似，相当于同样是做那样的数据可视化，但是分别制作了两种不同的展示途径，一种是基于Pyplot的静态图片展示，另一种是基于echart的动态网页展示。

使用说明（Quickstart）

1. 安装依赖
    - 当前模块的依赖写在 `requirements.txt`，建议使用虚拟环境。

2. 生成随机数据并绘制静态图
    - 运行 `scripts/demo.py` 将在 `outputs/` 下生成 `sample.json.gz`（数据）和 `sample_plot.png`（静态图）。

3. 启动网页端（FastAPI + ECharts）
    - 启动 `web/server.py`，浏览器访问 `http://127.0.0.1:8000/`，页面会调用 `/api/dataset` 接口并展示热力图与右侧横向柱状图。

数据存储格式（约定）
- 保存为 `.json.gz`（Gzip 压缩的 JSON），顶层包含：
  - `meta`: 数据集元信息（版本、创建时间、方块数量、时间跨度等）
  - `squares`: 列表，每个元素为一个方形区域的数据，包含：
     - `id`: 方形区域编号
     - `x_sec`, `y_sec`: 每个点在 X/Y 轴上的“秒”坐标
     - `value`: 每个点的取值，范围 [0, 1]

可视化说明
- 热力图：由散点平滑化得到，仅显示右下三角区域；颜色条放置在左上空白区域；横轴（下方）和纵轴（右侧）以 HH:MM:SS 显示。
- 右侧横向柱状图：按“离对角线距离”分段统计平均值，自上而下依次为距离由大到小；右侧标注百分比（如 87%）。
